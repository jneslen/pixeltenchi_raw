
・SRAMのチート又は書き換えについて
　SRAM(アドレス 0x10000以上)のチート又は書き換えを実行するときはまず、セーブデータのバックアップをして下　さい。
　ゲームによっては、SRAM内容をチェックしているので、特にロード、セーブ時は危険です。
　その時は一時的に数値固定をしないようにした方がいいと思います。
　最悪の場合セーブデータが使用出来なくなる恐れがあります。

・ROM読み込み時にROMのヘッダーとnestoyのデータを比較して、違うときにはダイアログが出ます。
　"はい"を選ぶとデータベースのデータが使用されます。



・チートコード

アドレス-サイズ-数値-コメント(無くてもいい)

		トレイナー
		画面モード（０＝２画面モード，１＝１画面モード）

        データベースと一致するかを表示します。(＝なら一致)

NAME		データベースのゲーム名
C		国
MAKER		データベースのメーカー




-- FDSのデータ表示はNESと違います。

P CRC		ブロック内容(0:データ 1:キャラクタデータ 2:ネームテーブルデータ)
C CRC		実際のブロック数
CHECK		チェック結果表示
HEAD		未使用
DBInfo		未使用

MAKER		データベースのメーカー
R=D		未使用

---------.chtファイル--------- VER 0

address		size
0		4byte		0
4		4byte		数値固定の総数
4 - 0x0f	8byte		未使用

0x10からは
{
コメント	17byte
アドレス	4byte
数値		1byte
チェック	1byte
数値のタイプ	1byte		(0=10進数、1=16進数)
}
の繰り返しです。


---------.chtファイル--------- VER 1

address		size
0		4byte		1
4		4byte		数値固定の総数
4 - 0x0f	8byte		未使用

0x10からは
{
コメント	37byte
アドレス	4byte
数値		4byte
チェック	1byte
数値のタイプ	1byte		(0=10進数、1=16進数)
数値のサイズ	1byte		(0=1Byte, 1=2Byte, 2= 4Byte , 3= 3Byte)
}
の繰り返しです。




-------.cdtファイル----------
address		size
0		4byte		バージョン、今は取り敢えず0です。
4		4byte		セーブデータフラグ
	・bit 0		保存１
	・bit 1		保存２
	・bit 2		保存３
	・bit 3		検索結果

	・1ならセーブデータに含まれています。

4 - 0x0f	8byte		未使用
0x10からはデータがあります。
・保存１
・保存２
・保存３
・検索結果
（フラグ0x800バイト、前のデータ0x800バイトです）
